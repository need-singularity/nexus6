## 3. 24-instruction category summary

### 3.1 funct3 x variant matrix (6 x 4 = 24 = sigma*phi = n*tau)

```
funct3  primitive    var=00      var=01      var=10      var=11
000     Omega_1 GEMM    gemm.m      gemm.a      gemm.tc     gemm.bf16
001     Omega_2 SOFTMAX sm.exp      sm.max      sm.sub      sm.norm
010     Omega_3 TOPK    tk.sort     tk.pick     tk.merge    tk.idx
011     Omega_4 GATE    gt.mul      gt.and      gt.or       gt.xor
100     Omega_5 REDUCE  rd.sum      rd.max      rd.min      rd.mean
101     Omega_6 CONV6   conv.3x3    conv.1x1    conv.dw     conv.pw
110     RSVD            -           -           -           -
111     RSVD            -           -           -           -
```

### 3.2 Category-wise meaning

| funct3 | Primitive | Category summary | Mean cycles |
|--------|-----------|------------------|------------:|
| 000 | GEMM | 12x12 master/accumulate/async TC/BF16 4 variants | sigma=12 |
| 001 | SOFTMAX | Phi6 exp -> max -> sub -> norm chain 4 stages | phi=2 |
| 010 | TOPK | sort/top-phi/merge/idx extraction (MoE router) | phi^2=4 |
| 011 | GATE | mul/AND/OR/XOR sparse-mask 4 variants | mu=1 |
| 100 | REDUCE | sum/max/min/mean tau=4 reduce | tau=4 |
| 101 | CONV6 | 3x3/1x1/depth-wise/point-wise 4 kernels | n=6 |

Overall average CPI ~ 29/6 ~ phi^2 = 4; the P-ISA-1 target (CPI <= phi=2) is a draft target, to be achieved via Phase 3 pipeline parallelism.

### 3.3 24 n=6 formula coherences

- **Omega_1 GEMM** 4 variants: sigma^2 = n*tau*phi^2 = 144 MAC, BF16 exponent sigma-tau=8
- **Omega_2 SOFTMAX** 4 variants: Phi6 (x^2-x+1) phi=2 cycles, Egyptian reciprocal LUT
- **Omega_3 TOPK** 4 variants: top-phi=2 MoE, tier-A n=6 slots
- **Omega_4 GATE** 4 variants: Boltzmann 1/e, Mobius mu(6)=1, Mertens, radical
- **Omega_5 REDUCE** 4 variants: tau=4 reduce tree depth=phi=2, mean = shift-tau-bit
- **Omega_6 CONV6** 4 variants: n=6 taps, tau=4 channels, sigma=12 length, sigma*tau=48 MAC depth-wise

## 4. Bit-encoding summary

RISC-V R-Type 32-bit / opcode = 0x0B (custom-0) fixed:

```
 31       25 24   20 19   15 14  12 11    7 6      0
+----------+-------+-------+------+-------+--------+
|  funct7  |  rs2  |  rs1  |funct3|  rd   | opcode |
|  7 bit   | 5 bit | 5 bit |3 bit | 5 bit | 7 bit  |
+----------+-------+-------+------+-------+--------+

funct7 = [reserved:4 | n6:1 | variant:2]
       = 0b0000_1vv  (v=00~11 = tau selection)
Base funct7 = 0x04, 0x05, 0x06, 0x07

funct3 in {000,001,010,011,100,101} -> 6 = n primitives used
funct3 in {110,111}                -> 2 reserved (BT-380 meta promotion)
```

- Total custom-0 space = 24 (current) + funct7 upper 4 bits x funct3 2 reserved = 384 = sigma^2 + sigma*tau*phi expansion room.
- `rsvd` field (funct7[6:3]=0000) reserved for Phase 4 Xn6-MEM/DMA.

## 5. GCC intrinsic samples (3)

### Sample 1 — Phi6 softmax chain (Omega_2 4-instruction fuse)

```c
#include <xn6intrin.h>

// Apply Phi6-based softmax to a 12-element vector
// Conventional softmax 14 cycles -> Xn6 4 instructions x mean phi=2 = 8 cycles (1.75x)
// + Phi6 approximation => GELU 7x savings -> total effective ~14x
void softmax_phi6_vec12(float* dst, const float* src) {
    float tmp[12];
    float max_val;

    __builtin_xn6_sm_exp (tmp, src);              // sm.exp   (phi=2)
    __builtin_xn6_sm_max (&max_val, tmp);         // sm.max   (phi^2=4)
    __builtin_xn6_sm_sub (tmp, tmp);              // sm.sub   (1)
    __builtin_xn6_sm_norm(dst, tmp);              // sm.norm  (phi=2)
}
```

### Sample 2 — 12x12 BF16 GEMM (Omega_1 training primary)

```c
#include <xn6intrin.h>

// Core of LLM training — BF16 precision 12x12 matmul 1 tile
// sigma=12 cycles for sigma^2=144 MACs, ~12x vs 256 scalar multiplies
void bf16_gemm_tile(__bf16* c, const __bf16* a, const __bf16* b) {
    // 1 instruction = 1 tile = 144 MACs (sigma^2)
    __builtin_xn6_gemm_bf16(c, a, b);
}

// Large-matmul call: N x N tiling
void bf16_matmul_large(__bf16* C, const __bf16* A, const __bf16* B, int N) {
    for (int i = 0; i < N; i += 12) {
        for (int j = 0; j < N; j += 12) {
            __builtin_xn6_gemm_m(&C[i*N+j], &A[i*N], &B[j]);        // first tile
            for (int k = 12; k < N; k += 12) {
                __builtin_xn6_gemm_a(&C[i*N+j], &A[i*N+k], &B[k*N+j]);  // accumulate
            }
        }
    }
}
```

### Sample 3 — MoE router (Omega_3 tk.pick + Omega_4 gt.mul fuse)

```c
#include <xn6intrin.h>

// Egyptian MoE router — select top-phi=2 from sigma=12 expert scores then
// apply gating mask. Conventional softmax+topk 14+10 -> Xn6 2+1 = 3 cycles
void egyptian_moe_dispatch(float* token_out,
                           const float* token_in,
                           const float* expert_scores,  // sigma=12
                           float* expert_cache) {
    int top2_idx[2];
    __builtin_xn6_tk_pick(top2_idx, expert_scores);   // tk.pick (phi=2)

    // Only 2 experts active (Boltzmann gate)
    float mask[12] = {0};
    mask[top2_idx[0]] = 1.0f;
    mask[top2_idx[1]] = 1.0f;

    __builtin_xn6_gt_mul(token_out, expert_cache, mask);  // gt.mul (mu=1)
}
```

## 6. hexa-lang `@xn6` attr style (R12 AI-NATIVE)

```
@xn6(op="gemm_m")
fn matmul12(a: tensor(12,12), b: tensor(12,12)) -> tensor(12,12) {
    return gemm(a, b)
}
```

When the compiler sees an `@xn6` attr it automatically inserts the corresponding `__builtin_xn6_*` intrinsic -> users can use all 24 instructions with **0 lines of manual asm**. R12 compliant.
