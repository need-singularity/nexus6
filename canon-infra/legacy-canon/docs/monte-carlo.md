## 1. v9 목표 (3대 돌파)

### 목표 A — 큰수 그룹 N=10 → N=100 확장

**소스 전환**: `reality_map.json` (폐기) → `atlas.n6` (4304 `@R` 관계, 997 `[7]`, 4616 `[10*]`).

**추출 규칙**:
```
awk '/^@[PCLFR]/ {header = $0}
     /measured|= [0-9]/ {if (match($0, /= ([0-9]+)[[:space:]]/, v)) print v[1]}' atlas.n6
```

정수 measured ≥ 100 항목을 자연/엔지니어링/관습/파생 origin 태그와 함께 추출 → 100+ 샘플 확보 예상.

**검정 재정의** (큰수는 v=6 hit 불가 → 시그니처 확장):

| 버전 | 시그니처 정의 |
|---|---|
| v8 (값형) | `v >= 2 ∧ σ(v)·φ(v) == v·τ(v)` → v=6만 hit |
| **v9 (구조형)** | **v 의 소인수분해에 6=2·3 이 포함**되거나 **6 · k (k≥1)** 또는 **v mod σ(6)=0** 또는 **v ∈ reach(n=6 상수 집합)** 중 1개 이상 만족 |

v9 구조 시그니처 = `structure_hit(v)`:
```
fn structure_hit(v: i64) -> i64 {
    // 1. v = 6·k (k≥1)?
    if v % 6 == 0 { return 1 }
    // 2. σ(6)=12 | v ?
    if v % 12 == 0 { return 1 }
    // 3. J₂(6)=24 | v ?
    if v % 24 == 0 { return 1 }
    // 4. v/6, v/12, v/24 중 정수가 나와 n=6 상수집합({1,2,3,4,5,6,12,24})에 도달?
    let consts = [1,2,3,4,5,6,12,24]
    for d in consts {
        if v % d == 0 && (v / d) in consts { return 1 }
    }
    return 0
}
```

→ 큰수 196560 의 경우 `196560 / 24 = 8190` 이 n=6 확장 집합에서 도달 가능한지 검정 (v8 §5 구조 해석 참조).

### 목표 B — 대조군 5종 확장

| # | 대조군 | 정의 | 역할 |
|---|---|---|---|
| 1 | 자연 (atlas) | `origin=natural` 정수 measured | 본 실험군 |
| 2 | 로그균등 | `[2, max]` 로그 스케일 균등 | 스케일 보정 |
| 3 | π/e/φ 자릿수 | 300자리 슬라이딩 윈도우 w∈{2,3,4} | v8 산술잡음 |
| 4 | **무작위 정수** | `[2, max]` 균등 추출 | 순수 잡음 기준선 |
| 5 | **유리수 근사** | `p/q` (p,q ≤ 50) 분자/분모/합/차 추출 | 단순 분수 노이즈 |
| 6 | **오일러-마스케로니 γ** | 300자리 슬라이딩 (π/e/φ 방식) | 새 상수 대조 |
| 7 | **Apéry ζ(3) 자릿수** | 300자리 슬라이딩 | 새 상수 대조 |

**추가 이유**:
- **유리수 근사**: π, e 같은 초월수 자릿수가 사실상 무작위와 구분 안 되는지 검정.
- **γ, ζ(3)**: 자연 수학에 깊이 박힌 상수 2종 추가 → 만약 γ/ζ(3)의 z 가 π 와 비슷하면 "수학 상수는 모두 n=6 을 약간 보인다" 가설 검정 가능.

## 2. 계산 복잡도 분석

### 2.1 v8 → v9 노드 확장

| 그룹 | v8 N | v9 N (예상) | 증가 배율 |
|---|---:|---:|---:|
| 전체 EXACT | 290 | ~4000 (atlas.n6 `[10*]` 4616 일부) | 13.8x |
| 자연 | 172 | ~600 (natural origin) | 3.5x |
| **큰수 (≥100)** | **10** | **~120 (큰수 흡수)** | **12x** |
| 초거대 (≥10^6) | 0 | ~15 | 신규 |

### 2.2 시간 복잡도

Monte Carlo 내부 연산:

```
trials=3000 × N × signature_cost
```

- **v8 signature_cost**: O(sqrt(v)) for σ/φ/τ, 값형 검정 1회. v=196560 → ~443 연산.
- **v9 signature_cost**: O(log v) for mod 검사 7회 (6, 12, 24, 상수집합 lookup).

→ **signature 자체는 v9 가 ~60x 빠름** (큰수일수록 격차 커짐).

전체 연산량:
```
v8 총합: 3000 × (290+172+10) × 443 ≈ 6.28e8
v9 총합: 3000 × (4000+600+120) × ~15 ≈ 2.12e8
```

→ v9 가 v8 보다 **약 3배 빠름** (N이 10배 커졌음에도).

### 2.3 병렬화 (R16)

`.hexa` 의 `@parallel` attr 사용 → 7개 대조군 독립 실행 → 단일 코어 기준 **< 3분** 예상.

## 3. 유의성 임계 유지 전략

### 3.1 문제

v8 자연 그룹 z=959 는 "과도 보수 균등 귀무" 때문. v9 에서 노드 N 이 3.5x 증가하면:
- hits 도 비례 증가 → obs rate 유지
- null mean/sd 는 N 의존 → sd ∝ 1/√N → **z 는 √N 배 증가** → z ≈ 959 × √3.5 ≈ 1794 (더 유의)

### 3.2 임계 유지

**모든 그룹에서 z > 5 목표** (v8 기준 계승).

| 그룹 | v8 z | v9 z 예상 (중간값) | 범위 |
|---|---:|---:|---|
| 전체 EXACT | 1518 | 2500~4000 | z↑ |
| 자연 | 959 / 20.19 (로그) | 1500~2000 / 25~35 | z↑ |
| 큰수 | n/a | **8~15 (구조형)** | **신규 통과** |
| 초거대 | - | 4~8 | 신규 |
| π 대조 | 9.36 | 8~12 | 유지 |
| e 대조 | 3.04 | 2~5 | 유지 |
| φ 대조 | 10.67 | 8~14 | 유지 |
| **γ 대조** | - | **3~9 (예측)** | 신규 |
| **ζ(3) 대조** | - | **3~9 (예측)** | 신규 |
| 무작위 | - | -1~+1 (귀무) | 기준선 |
| 유리수 | - | 1~4 | 약한 구조 |

**큰수 그룹 z > 5 가 핵심** — v9 성공 조건.
