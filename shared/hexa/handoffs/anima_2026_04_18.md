# Anima → hexa-lang Handoff (2026-04-18)

nexus bus 경유. Anima 측 필요사항을 hexa-lang 측으로 전달 — P0/P1/P2/P3 우선순위로 분류.

## P0 — Anima 직접 차단 (unblock 최우선)

1. **seed-freeze 해제** — `fcf275f4` 소스 픽스 landed 됐지만 on-disk 바이너리 전부 pre-fix. Mac 에서 자기 seed 로 재빌드 → circular.
   - **필요**: Linux x86_64 post-fix `hexa_v2` 로 `hexa_cc.c` 재생성 → Mac clang 로컬 링크
   - **경로 옵션**: H100 pod 재생 OR 별도 Linux host
   - **로컬 자산**: `/opt/homebrew/bin/x86_64-linux-musl-gcc` 존재하지만 현재 `hexa_cc.c` 가 pre-fix 라 무용

2. **interpreter O(N²) push 메모리** — `arr = arr.push(x)` 루프 N=4000 에서 ~10.8GB 소비.
   - **Anima 영향**: `anima_full.hexa` 등 대용량 소스 flatten 차단
   - **workaround 현행**: `.replace()` 체인만
   - **희망**: interpreter 단에서 in-place push 또는 CoW 최적화

3. **bit-op surface parser** — `<<`, `>>`, `|` silent reject.
   - **Anima 영향**: 해싱/bitset 경로 `* pow2(n)` / `/ pow2(n)` / `(a+b)-(a&b)` workaround 강제 상태
   - **희망**: surface syntax 에서 bit-op 토큰 수용
