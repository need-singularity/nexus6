nexus-cli

외부 프로젝트(hive 등) 전용 단일 CLI 진입점. nexus 의 thinking/smash/free/lens/atlas/discovery/roadmap/bus/status 엔진을 subprocess 경유로 사용.

설치

export NEXUS="$HOME/Dev/nexus"
export PATH="$NEXUS/shared/bin:$PATH"

# 확인
nexus-cli version
nexus-cli --help

서브커맨드

글로벌 플래그

flagdesc
--json기계 파싱용 JSON stdout (default: pretty)
--caller <name>외부 호출자 식별 (audit log 기록)
--help, -h도움말

사용 예시

# thinking
nexus-cli thinking --query "n=6 완전수의 양자 대칭성" --json --caller hive

# smash (blowup)
nexus-cli smash --seed "ramanujan partition identity" --depth 5 --json

# free (DFS)
nexus-cli free --seed "modular form moonshine" --dfs 4

# lens
nexus-cli lens --query "quantum" --top 10 --json

# atlas
nexus-cli atlas search "sigma" --limit 5 --json
nexus-cli atlas append --node '{"id":"TEST-1","name":"test","grade":"CONJECTURE"}' --json

# discovery
nexus-cli discovery query "10" --json

# roadmap
nexus-cli roadmap status --project nexus --json

# bus
nexus-cli bus tail --limit 10 --json
nexus-cli bus publish --topic "test" --payload '{"msg":"hello"}' --caller hive

# status
nexus-cli status --json

exit codes

codemeaning
0성공
1실행 오류 (엔진 실패, 내부 오류)
2인자 오류 (unknown subcmd, missing flag)
3gate 거부 (harness violation, cmd_gate REJECT)

보안

seed/query 입력에서 shell metacharacter (; | & $ ` \ " ' < > ( ) { } ! # ~ 개행) 자동 strip. --caller 값은 audit log (shared/logs/nexus_cli.log)에 JSONL 형태로 기록.

hive 연동

# hive.json caps 에서 invoke 패턴:
# "invoke": ["nexus-cli", "thinking", "--query", "{query}", "--json", "--caller", "hive"]

# hive subprocess 호출 예시 (hexa)
let out = exec("nexus-cli smash --seed 'quantum gravity' --json --caller hive")
let result = json_parse(out)
